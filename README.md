# QA Engineering Portfolio — Svilen Borisov

Hands-on QA work demonstrating manual test design, exploratory testing, bug reporting,
and API test automation. Each folder is a self-contained project with its own README.

## Projects

| Project | What it demonstrates |
|---------|----------------------|
| [**api-testing-inv-bg**](./api-testing-inv-bg) | **Risk-based API testing** (Postman) of a real invoicing API — 9-request self-cleaning suite, 16 assertions, secure token handling. **Found a real API bug** (REST-semantics violation) with confirmed root cause. |
| [**api-testing-postman**](./api-testing-postman) | **API testing + CI/CD** — Postman collection, environments/variables, auth flow, negative tests, **Newman + GitHub Actions** pipeline, bug report, test strategy. |
| [**bug-reports-saucedemo**](./bug-reports-saucedemo) | **Bug reporting & triage** — 7 professional bug reports with reproduction steps and **justified Severity vs Priority**; real-vs-by-design triage; logged in Jira. |
| [**qa-test-design-demoqa**](./qa-test-design-demoqa) | **Test design (EP + BVA)** — 9 test cases authored and executed in **Jira (Kanban)** with evidence. |
| [**exploratory-testing-academybugs**](./exploratory-testing-academybugs) | **Exploratory testing (SBTM)** — charter-driven, time-boxed session; 25-defect session sheet with severity triage; professional debrief. |

## Skills shown
- Test design: Equivalence Partitioning, Boundary Value Analysis
- Exploratory testing: SBTM, charters, heuristics (SFDPOT)
- Bug reporting: repro steps, Severity vs Priority, triage
- API testing: REST, Postman, assertions (status/format/data), risk-based prioritisation
- Automation/CI: Newman, GitHub Actions
- Tools: Postman, Jira, Git/GitHub
